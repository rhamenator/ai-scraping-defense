Quick Start

  1. Copy everything into repo root; commit & push.
  2. GitHub → Settings → Environments:
    • staging (no approval): secrets GCP_PROJECT, GCP_REGION, GKE_CLUSTER, GCP_SA_KEY (JSON), STAGING_BASE_URL (e.g. https://staging.example.com).
    • production (approval‑gated): same GCP secrets + PROD_BASE_URL.
  3. DNS: staging.example.com and *.staging.example.com → your ingress LB.
  4. Protect main; require checks (CI, Security Audit, Regression E2E).
